# 轨迹显示模式切换功能实现报告

> **功能**: 在轨迹界面右上角添加工具栏按钮，支持详细/精简两种显示模式切换  
> **实现日期**: 2026-04-26  
> **默认模式**: 详细模式（显示所有点）

---

## 🎯 功能概述

### 用户需求
- 数据量不大（每天约 50 个点），但希望有灵活性
- 默认显示详细信息（所有轨迹点）
- 可切换到精简模式（只显示关键点：起点、终点、停留点）
- 通过右上角工具栏按钮快速切换

### 实现方案
在轨迹界面右上角工具栏添加第四个按钮，点击后在**详细模式**和**精简模式**之间切换。

### 3. TrackActivity 代码修改

#### 3.1 添加成员变量

```java
private ImageButton toggleDisplayModeBtn;
private TextView displayModeText;
private boolean isSimpleMode = false; // 显示模式：false=详细模式，true=精简模式
```

#### 3.2 初始化按钮

在 `initToolbar()` 方法中添加：

```java
toggleDisplayModeBtn = findViewById(R.id.toggle_display_mode_btn);
displayModeText = findViewById(R.id.display_mode_text);

toggleDisplayModeBtn.setOnClickListener(new View.OnClickListener() {
    @Override
    public void onClick(View v) {
        toggleDisplayMode();
    }
});
```

#### 3.3 实现切换逻辑

新增 `toggleDisplayMode()` 方法：

```java
private void toggleDisplayMode() {
    isSimpleMode = !isSimpleMode;
    
    // 更新按钮文本
    if (displayModeText != null) {
        displayModeText.setText(isSimpleMode ? R.string.simple : R.string.detailed);
    }
    
    // 重新渲染轨迹
    if (!allLocationRecords.isEmpty()) {
        clearTrack();
        renderTrack(allLocationRecords);
    }
    
    String modeText = isSimpleMode ? getString(R.string.display_mode_simple) : getString(R.string.display_mode_detailed);
    Toast.makeText(this, modeText, Toast.LENGTH_SHORT).show();
    
    Log.d(TAG, "Display mode changed to: " + (isSimpleMode ? "simple" : "detailed"));
}
```

#### 3.4 修改渲染逻辑

在 `renderTrack()` 方法中，修改 Marker 创建逻辑：

**修改前**:
```java
// 如果不是停留点，且不是起点或终点，则跳过
if (!stayPoint.isStayPoint() && i != 0 && i != stayPoints.size() - 1) {
    Log.d(TAG, "Skipping ordinary point " + i);
    continue;
}
```

**修改后**:
```java
// 精简模式：如果不是停留点，且不是起点或终点，则跳过
if (isSimpleMode && !stayPoint.isStayPoint() && i != 0 && i != stayPoints.size() - 1) {
    Log.d(TAG, "Skipping ordinary point " + i + " (simple mode)");
    continue;
}
```

**关键变化**: 添加了 `isSimpleMode` 条件判断，只在精简模式下跳过普通轨迹点。

## 🔧 技术细节

### 1. 状态管理

使用布尔变量 `isSimpleMode` 跟踪当前显示模式：
- `false`: 详细模式（默认）
- `true`: 精简模式

### 2. 动态渲染

切换模式时：
1. 调用 `clearTrack()` 清除现有 Marker 和 Polyline
2. 调用 `renderTrack(allLocationRecords)` 重新渲染
3. `renderTrack()` 根据 `isSimpleMode` 决定是否跳过普通轨迹点

### 3. 性能优化

- **清除操作**: `clearTrack()` 会移除所有 Marker 并清空列表
- **重新渲染**: 只创建必要的 Marker，避免内存泄漏
- **日志记录**: 每次切换都会记录日志，方便调试

## 💡 扩展建议

### 1. 保存用户偏好

**问题**: 每次打开轨迹界面都是详细模式，用户可能需要每次都手动切换。

**解决方案**: 使用 SharedPreferences 保存用户的显示模式偏好

```java
// 保存偏好
SharedPreferences prefs = getSharedPreferences("track_settings", MODE_PRIVATE);
prefs.edit().putBoolean("simple_mode", isSimpleMode).apply();

// 读取偏好
boolean savedMode = prefs.getBoolean("simple_mode", false);
isSimpleMode = savedMode;
```

### 2. 自定义精简程度

**问题**: 有些用户可能想要中等程度的显示（例如显示每 10 个点）。

**解决方案**: 提供三种模式
- 详细模式：显示所有点
- 中等模式：每 N 个点显示一个
- 精简模式：只显示关键点

### 3. 动画效果

**问题**: 切换时 Marker 突然出现/消失，体验不够流畅。

**解决方案**: 添加淡入淡出动画

```java
marker.setAlpha(0f);
marker.animate().alpha(1f).setDuration(300).start();
```
