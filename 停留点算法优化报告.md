# 停留点算法优化报告

> **优化日期**: 2026-04-26  
> **优化目标**: 平衡轨迹详细度和 GPS 漂移抗干扰能力  
> **核心策略**: 距离阈值 + 时间阈值 + 最小点数三重判断

---

## 🎯 优化概述

### 用户需求

1. ✅ **轨迹尽可能详细** - 看到更多的轨迹信息
2. ✅ **减少 GPS 漂移影响** - 避免误判停留点
3. ✅ **适合开车场景** - 等红绿灯、堵车等短暂停留也能识别

### 优化前的问题

| 问题 | 原因 | 影响 |
|------|------|------|
| **停留点过多** | 距离阈值 20m 太严格 | GPS 抖动被误判为移动 |
| **短暂停留误判** | 时间阈值 60s 太短 | 等红绿灯也被判定为停留点 |
| **数据可靠性低** | 无最小点数要求 | 单个异常点就能形成停留点 |

---

## ✅ 已实施的优化

### 优化 1: 调整距离阈值为 30 米

**修改前**:
```java
private static final double STAY_DISTANCE_THRESHOLD = 20.0; // 20 米
```

**修改后**:
```java
private static final double STAY_DISTANCE_THRESHOLD = 30.0; // 30 米，平衡 GPS 精度和停留点识别
```

**理由**:
- ✅ **覆盖 GPS 误差** - 大多数 GPS 设备精度在 10-30 米
- ✅ **保留详细信息** - 不会过度合并 nearby 停留点
- ✅ **抗漂移能力强** - 减少因 GPS 抖动导致的误判
- ✅ **适合开车场景** - 等红绿灯、堵车时的短暂停留也能识别

---

### 优化 2: 增强停留点判断条件

**修改前**:
```java
public boolean isStayPoint() {
    return stayDuration >= 60000; // 仅判断时间 >= 60 秒
}
```

**修改后**:
```java
public boolean isStayPoint() {
    // 停留点判断条件：
    // 1. 停留时间 >= 2 分钟（减少 GPS 漂移和短暂停留的误判）
    // 2. 至少有 3 个采样点（确保数据可靠性）
    return stayDuration >= 120000 && mergedRecords.size() >= 3;
}
```

**理由**:
- ✅ **时间阈值提升到 2 分钟** - 过滤等红绿灯、临时停车等短暂停留
- ✅ **增加最小点数要求** - 确保有足够的采样数据，提高可靠性
- ✅ **双重保护** - 时间和点数都满足才判定为停留点

## 🛡️ GPS 漂移防护机制

### 三层防护

#### 第 1 层: 异常点过滤（预处理）

```java
// 在 processStayPoints 之前调用 filterAbnormalPoints
List<LocationRecord> filteredRecords = filterAbnormalPoints(locationRecords);
```

**作用**:
- ✅ 过滤 GPS 跳变（60秒内 > 2000m）
- ✅ 过滤速度异常（> 200 km/h）
- ✅ 过滤时间倒流
- ✅ 过滤无效坐标

**效果**: 进入停留点算法的数据已经是"干净"的

---

#### 第 2 层: 距离阈值（30 米）

```java
if (distance <= STAY_DISTANCE_THRESHOLD) {  // 30m
    mergedRecords.add(nextRecord);
}
```

**作用**:
- ✅ 允许 GPS 在 30m 范围内抖动
- ✅ 不会因为小范围漂移而分裂停留点
- ✅ 聚类时动态更新中心点，进一步平滑

---

#### 第 3 层: 时间和点数双重判断

```java
public boolean isStayPoint() {
    return stayDuration >= 120000 && mergedRecords.size() >= 3;
}
```

**作用**:
- ✅ 时间 >= 2 分钟：过滤短暂停留
- ✅ 点数 >= 3 个：确保数据可靠性
- ✅ 双重保护：必须同时满足两个条件

---

### 防护效果示例

**场景: 室内 GPS 漂移 + 短暂外出**

```
原始数据:
10:00:00  A(39.9, 116.4)     - 室内
10:00:30  B(39.9001, 116.4001)  - GPS 漂移 15m
10:01:00  C(39.8999, 116.3999)  - GPS 漂移 18m
10:01:30  D(39.9002, 116.4002)  - GPS 漂移 25m
10:02:00  E(39.91, 116.41)      - 外出 1500m
10:02:30  F(39.91, 116.41)      - 室外
10:03:00  G(39.91, 116.41)      - 室外
10:03:30  H(39.91, 116.41)      - 室外
10:04:00  I(39.9, 116.4)        - 返回室内
10:04:30  J(39.9001, 116.4001)  - GPS 漂移 15m

处理过程:

第 1 层: 异常点过滤
- 无异常点，全部保留 ✅

第 2 层: 距离聚类（30m）
- A, B, C, D → 聚类 1（室内，4 个点，2 分钟）
- E, F, G, H → 聚类 2（室外，4 个点，2 分钟）
- I, J → 聚类 3（室内，2 个点，30 秒）

第 3 层: 停留点判断
- 聚类 1: 时间=120s ≥ 120s ✅, 点数=4 ≥ 3 ✅ → 是停留点 ✅
- 聚类 2: 时间=120s ≥ 120s ✅, 点数=4 ≥ 3 ✅ → 是停留点 ✅
- 聚类 3: 时间=30s < 120s ❌ → 不是停留点 ✅

最终结果:
- 室内停留 2 分钟（A-D）✅
- 室外停留 2 分钟（E-H）✅
- 返回后的短暂停留（I-J）被过滤 ✅
```
