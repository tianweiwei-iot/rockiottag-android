# RockiotTag - 定位精灵App详细开发计划

## 项目概述

RockiotTag是一个基于BLE（蓝牙低功耗）和GPS定位的精灵标签定位应用，参考华为定位精灵tag和小米tag的功能，通过蓝牙直接连接和众包网络实现设备的精确定位和追踪。

## 核心功能需求

1. **设备绑定与管理**：扫描、绑定、管理蓝牙定位标签
2. **近距离定位**：通过蓝牙直接连接获取tag的位置信息
3. **远距离众包定位**：利用其他用户的设备扫描到tag并共享位置信息
4. **地图显示**：在地图上显示tag的实时位置和历史轨迹
5. **后台追踪**：在后台持续扫描和更新tag位置
6. **通知提醒**：当tag超出安全范围或长时间未更新时发送提醒

## 技术架构

### 核心模块

1. **BLE管理模块**：负责设备扫描、绑定、连接和数据传输（包括电量信息获取）
2. **定位模块**：负责位置计算、融合和过滤
3. **地图模块**：负责地图显示、标记、轨迹和信息展示（位置、电量、时间）
4. **众包模块**：负责数据上传、下载和同步
5. **后台服务模块**：负责后台扫描和定位
6. **通知模块**：负责安全范围提醒和状态通知
7. **数据存储模块**：负责本地数据存储和管理（包括电量数据）
8. **用户界面模块**：负责用户交互和显示（包括底部信息栏）

### 技术栈

* **开发语言**: Java

* **地图SDK**: 高德地图SDK

* **蓝牙**: Android BLE API

* **存储**: SQLite

* **网络**: HttpURLConnection

* **UI**: Android原生UI组件

* **后台处理**: WorkManager, ForegroundService

* **二维码扫描**: ZXing或Google ML Kit

## 风险评估

1. **权限问题**: Android 10+对位置和蓝牙权限有更严格的要求
2. **后台限制**: Android 8+对后台服务有严格限制，可能影响后台扫描
3. **设备兼容性**: 不同设备的BLE性能差异较大
4. **网络稳定性**: 众包数据上传依赖网络，需要处理网络不稳定情况
5. **电量消耗**: 持续的BLE扫描和定位可能导致电量消耗过快
6. **定位精度**: 室内环境下BLE定位精度可能受到干扰

## 解决方案

1. **权限问题**: 使用最新的权限申请API，提供清晰的权限说明和引导
2. **后台限制**: 使用前台服务和WorkManager结合的方式，确保后台扫描
3. **设备兼容性**: 针对不同设备进行测试，优化扫描参数和连接逻辑
4. **网络稳定性**: 实现数据缓存和重试机制，确保数据上传和同步
5. **电量消耗**: 优化扫描频率和定位精度，使用低功耗模式
6. **定位精度**: 结合多种定位技术，使用卡尔曼滤波等算法提高精度
