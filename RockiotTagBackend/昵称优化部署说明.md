# 设备昵称优化部署说明

## 📋 修改内容

### 1. 后端代码修改

#### ✅ 已完成的修改

**文件1: `DataSyncService.java`**
- 位置: `RockiotTagBackend/src/main/java/com/rockiot/tag/service/DataSyncService.java`
- 修改: 同步设备数据时过滤供应商自动生成的数字昵称（`^0+\d+$`）
- 效果: 只有用户手动设置的有意义昵称才会被保存

**文件2: `DeviceController.java`**
- 位置: `RockiotTagBackend/src/main/java/com/rockiot/tag/controller/DeviceController.java`
- 修改: 更新设备昵称时应用相同的过滤规则
- 效果: 允许用户将昵称设置为空（NULL）

**文件3: `VendorApiService.java`**
- 位置: `RockiotTagBackend/src/main/java/com/rockiot/tag/service/VendorApiService.java`
- 修改: 绑定设备时，如果未指定昵称则不发送nickName字段
- 效果: 供应商API不会自动生成数字昵称

### 2. 数据库清理脚本

**文件: `cleanup_numeric_nicknames.sql`**
- 位置: `RockiotTagBackend/cleanup_numeric_nicknames.sql`
- 功能: 批量将现有的"000000"开头数字昵称更新为NULL

### 步骤4: 执行数据库清理脚本

#### 方法1: 直接上传SQL文件执行

```bash
# 在服务器上创建SQL文件
cat > /tmp/cleanup_numeric_nicknames.sql << 'EOF'
-- 查看将被影响的设备数量
SELECT COUNT(*) AS affected_devices FROM devices WHERE nick_name REGEXP '^0+[0-9]+$';

-- 查看将被影响的具体设备
SELECT id, device_num, nick_name, created_at FROM devices WHERE nick_name REGEXP '^0+[0-9]+$' ORDER BY created_at DESC;

-- 执行批量更新
UPDATE devices SET nick_name = NULL WHERE nick_name REGEXP '^0+[0-9]+$';

-- 验证更新结果
SELECT COUNT(*) AS remaining_numeric_nicknames FROM devices WHERE nick_name REGEXP '^0+[0-9]+$';
EOF

# 执行SQL脚本
mysql -u root -p rockiot_tag < /tmp/cleanup_numeric_nicknames.sql
```

#### 方法2: 逐条执行SQL命令

```sql
-- 1. 查看影响范围
SELECT COUNT(*) AS affected_devices FROM devices WHERE nick_name REGEXP '^0+[0-9]+$';

-- 2. 查看具体设备
SELECT id, device_num, nick_name FROM devices WHERE nick_name REGEXP '^0+[0-9]+$' LIMIT 20;

-- 3. 执行更新（确认无误后执行）
UPDATE devices SET nick_name = NULL WHERE nick_name REGEXP '^0+[0-9]+$';

-- 4. 验证结果
SELECT COUNT(*) AS remaining_numeric_nicknames FROM devices WHERE nick_name REGEXP '^0+[0-9]+$';
```

## ⚠️ 注意事项

1. **备份数据库**: 执行UPDATE前建议先备份
   ```bash
   mysqldump -u root -p rockiot_tag > backup_$(date +%Y%m%d).sql
   ```

2. **确认影响范围**: 先运行SELECT语句确认将被更新的设备数量

3. **回滚方案**: 如需回滚
   ```bash
   mysql -u root -p rockiot_tag < backup_20260426.sql
   ```

4. **Android App兼容性**: 
   - App端已有过滤逻辑（`!deviceInfo.nickName.matches("\\d+")`）
   - NULL值在JSON中会显示为null，App端会正确处理

5. **供应商API行为**:
   - 不发送nickName字段时，供应商可能仍会生成默认昵称
   - 但我们的同步逻辑会过滤掉这些数字昵称，保持数据库为NULL
