## 1. 数据同步逻辑优化

### 当前问题

**位置**: `TrackActivity.java` 第 867-876 行

```java
if (syncFromServer) {
    Log.d(TAG, "Syncing from server first...");
    Toast.makeText(this, R.string.loading_track_data, Toast.LENGTH_SHORT).show();
    syncTrackDataFromServerAndReload(
        selectedDevice.getDeviceNum() != null ? selectedDevice.getDeviceNum() : selectedDevice.getDeviceId(),
```

**问题分析**:
- 每次加载轨迹都会先从服务器同步数据，导致不必要的网络请求
- 如果本地已有数据且时间范围合理，应该优先使用本地缓存
- 频繁同步会增加服务器压力和流量消耗

### 优化建议

#### 方案 1: 智能缓存策略

```java
private void loadTrackData(final boolean forceSync) {
    if (selectedDevice == null) {
        Toast.makeText(this, R.string.please_select_device, Toast.LENGTH_SHORT).show();
        return;
    }

    clearTrack();

    final Calendar startTime = (Calendar) startDate.clone();
    final Calendar endTime = (Calendar) endDate.clone();

    // 先查询本地数据
    List<LocationRecord> localRecords = databaseHelper.getLocationRecords(
        selectedDevice.getDeviceId(),
        startTime.getTimeInMillis(),
        endTime.getTimeInMillis()
    );

    // 如果本地有数据且不是强制同步，检查数据新鲜度
    if (!forceSync && localRecords != null && !localRecords.isEmpty()) {
        long newestTimestamp = localRecords.get(localRecords.size() - 1).getTimestamp();
        long cacheAge = System.currentTimeMillis() - newestTimestamp;
        
        // 如果数据在 5 分钟内，直接使用本地缓存
        if (cacheAge < 300000) {
            Log.d(TAG, "Using cached data (age: " + cacheAge + "ms)");
            renderTrack(localRecords);
            return;
        }
    }

    // 否则从服务器同步
    Log.d(TAG, "Syncing from server...");
    Toast.makeText(this, R.string.loading_track_data, Toast.LENGTH_SHORT).show();
    syncTrackDataFromServerAndReload(
        selectedDevice.getDeviceNum() != null ? selectedDevice.getDeviceNum() : selectedDevice.getDeviceId(),
        startTime.getTimeInMillis(),
        endTime.getTimeInMillis()
    );
}
```

```java
private void syncTrackDataIncremental(String deviceNum, long startTime, long endTime) {

    LocationRecord latestLocal = databaseHelper.getLatestLocationRecord(deviceNum);
    long localLatestTime = latestLocal != null ? latestLocal.getTimestamp() : 0;
```

#### 方案 1: 轨迹点抽稀算法（推荐）

**Douglas-Peucker 算法实现**:

```java
/**
 * Douglas-Peucker 轨迹抽稀算法
 * @param points 原始轨迹点列表
 * @param epsilon 容差值（米），小于此距离的点将被过滤
 * @return 抽稀后的轨迹点
 */
private List<LatLng> douglasPeucker(List<LatLng> points, double epsilon) {
    if (points.size() < 3) {
        return new ArrayList<>(points);
    }
    
    // 找到距离首尾连线最远的点
    double maxDistance = 0;
    int index = 0;
    int end = points.size() - 1;
    
    for (int i = 1; i < end; i++) {
        double distance = perpendicularDistance(points.get(i), points.get(0), points.get(end));
        if (distance > maxDistance) {
            maxDistance = distance;
            index = i;
        }
    }
    
    // 如果最远距离大于容差，递归处理
    if (maxDistance > epsilon) {
        List<LatLng> left = douglasPeucker(points.subList(0, index + 1), epsilon);
        List<LatLng> right = douglasPeucker(points.subList(index, points.size()), epsilon);
        
        List<LatLng> result = new ArrayList<>(left.subList(0, left.size() - 1));
        result.addAll(right);
        return result;
    } else {
        // 否则只保留首尾点
        List<LatLng> result = new ArrayList<>();
        result.add(points.get(0));
        result.add(points.get(end));
        return result;
    }
}

/**
 * 计算点到线段的垂直距离
 */
private double perpendicularDistance(LatLng point, LatLng lineStart, LatLng lineEnd) {
    double dlat = lineEnd.latitude - lineStart.latitude;
    double dlon = lineEnd.longitude - lineStart.longitude;
    
    double mag = Math.sqrt(dlat * dlat + dlon * dlon);
    if (mag == 0) return 0;
    
    double u = ((point.latitude - lineStart.latitude) * dlat + 
                (point.longitude - lineStart.longitude) * dlon) / (mag * mag);
    
    if (u < 0) u = 0;
    if (u > 1) u = 1;
    
    double ix = lineStart.latitude + u * dlat;
    double iy = lineStart.longitude + u * dlon;
    
    return CoordinateUtils.calculateDistanceMeters(point.latitude, point.longitude, ix, iy);
}
```

```java
/**
 * 简单的轨迹点过滤（性能更好）
 * @param points 原始轨迹点
 * @param minDistance 最小距离间隔（米）
 * @param minTime 最小时间间隔（毫秒）
 */
private List<LatLng> filterTrackPoints(List<LatLng> points, double minDistance, long minTime) {
    if (points.isEmpty()) return points;
    
    List<LatLng> filtered = new ArrayList<>();
    filtered.add(points.get(0)); // 保留起点
    
    LatLng lastPoint = points.get(0);
    long lastTime = 0;
    
    for (int i = 1; i < points.size(); i++) {
        LatLng currentPoint = points.get(i);
        long currentTime = stayPoints.get(i).getArriveTime();
        
        double distance = CoordinateUtils.calculateDistanceMeters(
            lastPoint.latitude, lastPoint.longitude,
            currentPoint.latitude, currentPoint.longitude
        );
        
        long timeDiff = currentTime - lastTime;
        
        // 如果距离或时间间隔超过阈值，保留该点
        if (distance > minDistance || timeDiff > minTime) {
            filtered.add(currentPoint);
            lastPoint = currentPoint;
            lastTime = currentTime;
        }
    }
    
    filtered.add(points.get(points.size() - 1)); // 保留终点
    return filtered;
}
```

- GPS 在室内或高楼区域会产生跳变（如突然跳到几百米外又跳回来）
- 缺乏对时间间隔异常的检测

### 优化建议

#### 增强版异常点过滤器

```java
private List<LocationRecord> filterAbnormalPoints(List<LocationRecord> records) {
    if (records.size() < 2) return records;
    
    List<LocationRecord> filtered = new ArrayList<>();
    filtered.add(records.get(0));
    
    int filteredOut = 0;
    int gpsSpikeCount = 0;
    int speedAnomalyCount = 0;
    
    for (int i = 1; i < records.size(); i++) {
        LocationRecord prev = filtered.get(filtered.size() - 1);
        LocationRecord curr = records.get(i);
        
        double distance = CoordinateUtils.calculateDistanceMeters(
            prev.getLatitude(), prev.getLongitude(),
            curr.getLatitude(), curr.getLongitude()
        );
        
        double timeDiffHours = (curr.getTimestamp() - prev.getTimestamp()) / 3600000.0;
        double timeDiffSeconds = (curr.getTimestamp() - prev.getTimestamp()) / 1000.0;
        
        // 检查 1: GPS 漂移检测（短时间大距离跳变）
        if (timeDiffSeconds < 60 && distance > 500) {
            gpsSpikeCount++;
            Log.d(TAG, "Filtered GPS spike at index " + i + 
                  ": distance=" + String.format("%.0f", distance) + "m in " + 
                  String.format("%.0f", timeDiffSeconds) + "s");
            continue;
        }
        
        // 检查 2: 速度异常
        if (timeDiffHours > 0) {
            double speedKmh = (distance / 1000.0) / timeDiffHours;
            
            if (speedKmh > MAX_SPEED_KMH) {
                speedAnomalyCount++;
                Log.d(TAG, "Filtered speed anomaly at index " + i + 
                      ": speed=" + String.format("%.1f", speedKmh) + " km/h");
                continue;
            }
            
            // 警告：步行/骑行速度异常
            if (speedKmh > 50 && speedKmh <= MAX_SPEED_KMH) {
                Log.w(TAG, "Warning: High speed at index " + i + 
                      ": " + String.format("%.1f", speedKmh) + " km/h");
            }
        }
        
        // 检查 3: 时间倒流（设备时钟异常）
        if (curr.getTimestamp() < prev.getTimestamp()) {
            Log.w(TAG, "Filtered time reversal at index " + i);
            continue;
        }
        
        // 检查 4: 坐标无效（0,0 或其他异常值）
        if (Math.abs(curr.getLatitude()) < 0.0001 && Math.abs(curr.getLongitude()) < 0.0001) {
            Log.w(TAG, "Filtered invalid coordinates at index " + i);
            continue;
        }
        
        filtered.add(curr);
    }
    
    Log.d(TAG, "Filter results: total=" + records.size() + 
          ", kept=" + filtered.size() + 
          ", gpsSpikes=" + gpsSpikeCount + 
          ", speedAnomalies=" + speedAnomalyCount);
    
    return filtered;
}
```

### 预期效果

- ✅ 过滤 GPS 漂移，轨迹更平滑真实
- ✅ 避免异常点导致的距离计算错误
- ✅ 提高停留点识别准确度
- ✅ 更可靠的数据质量

## 4. 停留点算法优化

### 当前问题

**位置**: `TrackActivity.java` 第 78-79 行

```java

private static final double MAX_SPEED_KMH = 200.0;
```

**问题分析**:
- 20 米的距离阈值可能太小，GPS 误差可达 10-50 米
- 没有考虑最小停留时间（可能只是短暂停留）
- 简单距离判断容易受到 GPS 漂移影响

### 优化建议

#### 增强版停留点检测

```java
/**
 * 增强的停留点检测算法
 * @param records 轨迹记录列表
 * @param distanceThreshold 距离阈值（米）
 * @param minStayDuration 最小停留时间（毫秒）
 * @return 停留点列表
 */
private List<StayPoint> detectStayPoints(List<LocationRecord> records, 
                                         double distanceThreshold,
                                         long minStayDuration) {
    List<StayPoint> stayPoints = new ArrayList<>();
    
    if (records.isEmpty()) return stayPoints;
    
    int clusterStart = 0;
    LatLng clusterCenter = new LatLng(
        records.get(0).getLatitude(),
        records.get(0).getLongitude()
    );
    int clusterSize = 1;
    
    for (int i = 1; i < records.size(); i++) {
        LocationRecord record = records.get(i);
        LatLng currentPos = new LatLng(record.getLatitude(), record.getLongitude());
        
        double distance = CoordinateUtils.calculateDistanceMeters(
            clusterCenter.latitude, clusterCenter.longitude,
            currentPos.latitude, currentPos.longitude
        );
        
        if (distance <= distanceThreshold) {
            // 在聚类范围内，更新聚类中心
            clusterCenter = new LatLng(
                (clusterCenter.latitude * clusterSize + currentPos.latitude) / (clusterSize + 1),
                (clusterCenter.longitude * clusterSize + currentPos.longitude) / (clusterSize + 1)
            );
            clusterSize++;
        } else {
            // 离开聚类区域，检查是否形成停留点
            long stayDuration = records.get(i - 1).getTimestamp() - records.get(clusterStart).getTimestamp();
            
            if (stayDuration >= minStayDuration && clusterSize >= 3) {
                StayPoint stayPoint = new StayPoint(
                    clusterCenter.latitude,
                    clusterCenter.longitude,
                    records.get(clusterStart).getTimestamp(),
                    records.get(i - 1).getTimestamp(),
                    clusterStart,
                    clusterSize
                );
                stayPoints.add(stayPoint);
            }
            
            // 开始新的聚类
            clusterStart = i;
            clusterCenter = currentPos;
            clusterSize = 1;
        }
    }
    
    // 处理最后一个聚类
    if (clusterSize >= 3) {
        long stayDuration = records.get(records.size() - 1).getTimestamp() - records.get(clusterStart).getTimestamp();
        if (stayDuration >= minStayDuration) {
            StayPoint stayPoint = new StayPoint(
                clusterCenter.latitude,
                clusterCenter.longitude,
                records.get(clusterStart).getTimestamp(),
                records.get(records.size() - 1).getTimestamp(),
                clusterStart,
                clusterSize
            );
            stayPoints.add(stayPoint);
        }
    }
    
    Log.d(TAG, "Detected " + stayPoints.size() + " stay points from " + records.size() + " records");
    return stayPoints;
}
```

```java

```

```java
private void getAddressForLocation(final LatLng latLng) {

    new Thread(new Runnable() {
```

```java
private void syncTrackDataWithRetry(String deviceNum, long startTime, long endTime) {
    int maxRetries = 3;
    int retryCount = 0;
    long retryDelay = 1000; // 初始延迟 1 秒
    
    while (retryCount < maxRetries) {
        try {
            boolean success = syncFromServer(deviceNum, startTime, endTime);
            if (success) {
                return; // 成功，退出
            }
        } catch (Exception e) {
            Log.e(TAG, "Sync attempt " + (retryCount + 1) + " failed: " + e.getMessage());
        }
        
        retryCount++;
        if (retryCount < maxRetries) {
            try {
                Thread.sleep(retryDelay);
                retryDelay *= 2; // 指数退避
            } catch (InterruptedException e) {
                Thread.currentThread().interrupt();
                break;
            }
        }
    }
    
    // 所有重试失败
    runOnUiThread(new Runnable() {
        @Override
        public void run() {
            Toast.makeText(TrackActivity.this, "同步失败，请检查网络", Toast.LENGTH_LONG).show();
            
            // 提供重试按钮
            new AlertDialog.Builder(TrackActivity.this)
                .setTitle("同步失败")
                .setMessage("无法从服务器同步数据")
                .setPositiveButton("重试", (dialog, which) -> {
                    syncTrackDataWithRetry(deviceNum, startTime, endTime);
                })
                .setNegativeButton("使用本地数据", (dialog, which) -> {
                    loadTrackData(false);
                })
                .show();
        }
    });
}
```
