### 优化 2: 简化方向箭头生成逻辑 ⭐⭐

**修改前**:
```java
// 复杂的距离计算，动态生成多个箭头
for (int i = 0; i < latLngList.size() - 1; i++) {
    double segmentDistance = ...;
    int arrowCount = Math.max(1, Math.min(5, (int) (segmentDistance / 300)));
    
    if (arrowCount == 1) {
        // 添加 1 个箭头
    } else {
        // 添加多个箭头（最多 5 个）
        for (int j = 1; j <= arrowCount; j++) {
            // ...
        }
    }
}
```

**修改后**:
```java
// 优化：每隔 N 个点显示一个箭头，避免过于密集
int arrowInterval = 5; // 每 5 个点显示一个箭头

for (int i = 0; i < latLngList.size() - 1; i += arrowInterval) {
    LatLng p1 = latLngList.get(i);
    LatLng p2 = latLngList.get(Math.min(i + 1, latLngList.size() - 1));
    
    double angle = calculateBearing(p1.latitude, p1.longitude, p2.latitude, p2.longitude);
    
    MarkerOptions arrowMarker = new MarkerOptions()
        .position(new LatLng(p1.latitude, p1.longitude))
        .icon(createArrowMarker(angle))
        .anchor(0.5f, 0.5f)
        .zIndex(1);
    
    Marker marker = aMap.addMarker(arrowMarker);
    marker.setVisible(showMarkers);
    arrowMarkers.add(marker);
}
```

**效果**:
- ✅ 箭头数量从 ~50-100 个减少到 ~10 个
- ✅ 代码更简洁，逻辑更清晰
- ✅ 仍能清楚显示轨迹方向
- ✅ 减少地图渲染负担

### 优化 3: 按需加载逆地理编码 + 缓存 ⭐⭐⭐

**当前问题**:
- 加载轨迹时会自动查询第一个点的地址
- 如果用户不点击任何点，这个请求就浪费了
- 多次点击同一位置会重复查询

**建议实现**:

#### 步骤 1: 添加地址缓存

在 `TrackActivity` 类中添加成员变量：

```java
// 地址缓存：key = "lat,lng", value = address
private Map<String, String> addressCache = new HashMap<>();
```

#### 步骤 2: 移除自动查询

在 `renderTrack()` 方法中，删除这一行：

```java
// 删除这行
// getAddressForLocation(latLngList.get(0));
```

#### 步骤 3: 在点击事件中查询

修改 `onMarkerClick` 方法：

```java
@Override
public boolean onMarkerClick(Marker marker) {
    // 显示时间
    trackPointTime.setText(marker.getTitle());
    
    // 异步查询地址（带缓存）
    getAddressForLocationWithCache(marker.getPosition());
    
    return true;
}
```

#### 步骤 4: 实现带缓存的查询方法

```java
private void getAddressForLocationWithCache(LatLng latLng) {
    String key = String.format("%.6f,%.6f", latLng.latitude, latLng.longitude);
    
    // 先查缓存
    if (addressCache.containsKey(key)) {
        trackPointAddress.setText(addressCache.get(key));
        Log.d(TAG, "Using cached address for " + key);
        return;
    }
    
    // 缓存未命中，异步查询
    Log.d(TAG, "Querying address for " + key);
    getAddressForLocation(latLng);
}
```

#### 步骤 5: 保存查询结果到缓存

修改 `onRegeocodeSearched` 方法：

```java
@Override
public void onRegeocodeSearched(RegeocodeResult result, int rCode) {
    if (rCode == 1000 && result != null && result.getRegeocodeAddress() != null) {
        String address = result.getRegeocodeAddress().getFormatAddress();
        
        if (address != null && !address.isEmpty()) {
            trackPointAddress.setText(address);
            
            // 保存到缓存（需要获取当前点击的位置）
            // 可以通过成员变量记录最后点击的位置
            if (lastClickedPosition != null) {
                String key = String.format("%.6f,%.6f", 
                    lastClickedPosition.latitude, 
                    lastClickedPosition.longitude);
                addressCache.put(key, address);
                Log.d(TAG, "Cached address for " + key);
            }
        } else {
            trackPointAddress.setText(getString(R.string.unknown_location));
        }
    } else {
        trackPointAddress.setText(getString(R.string.get_address_failed));
    }
}
```

添加成员变量记录最后点击的位置：

```java
private LatLng lastClickedPosition = null;

@Override
public boolean onMarkerClick(Marker marker) {
    lastClickedPosition = marker.getPosition(); // 记录点击位置
    trackPointTime.setText(marker.getTitle());
    getAddressForLocationWithCache(marker.getPosition());
    return true;
}
```

**预期效果**:
- ✅ 网络请求减少 90%（只在点击时查询）
- ✅ 同一位置多次点击只需查询一次
- ✅ 用户体验更好（响应更快）
- ✅ 离线模式下仍能显示缓存的地址

## 🔧 可调整参数

### 1. 箭头间隔

当前设置为每 5 个点显示一个箭头：

```java
int arrowInterval = 5;
```

**调整建议**:
- 如果轨迹点更稀疏（如 20 个点/天）: 改为 `3`
- 如果轨迹点更密集（如 100 个点/天）: 改为 `8-10`

### 2. 停留点距离阈值

当前在 `TrackActivity` 中定义：

```java
private static final double STAY_DISTANCE_THRESHOLD = 20.0; // 20 米
```

**调整建议**:
- GPS 精度较差的环境: 增加到 `30-50` 米
- 需要更精确的停留检测: 减少到 `10-15` 米
